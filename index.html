MusicAI

MusicAI is a music notation software that was developed with a team of people in a software engineering course. The application works by obtaining user input to determine notes, rests, or any other musical element, and places them in the proper location on the sheet music depending on where the user clicks. The note is automatically adjusted to snap onto a readable position on the staff. The application determines whether or not you can place a note in a measure based on what is already present in the measure. This works by using linked lists to store note objects which are stored in measures (which is also linked in another linked list) which are stored in lines, etc. It is very efficient when searching and placing notes because of how the pointers of the linked lists are stored within these objects. Placing a note on the page takes O(log(n)) time.

• Wrote 4000 lines of C++ code using .NET framework to engrave sheet music following a specific set of rules of modern music notation

• Used vector translation for positioning document on zoom and user clicks relative to zoomed document

• Implemented a copy and paste feature as well as a multiple selection tool for user convenience

• Managed a team by tracking tasks on a Kanban board on Trello and tracked project version using Github

Fishopoly

Fishopoly is a fishing game that was developed using UNT's LARC game engine. The game is a 2.5 dimensional sidescroller that features procedural generation for creating and animating fish sprites and terrain. The fish sprites are created through cellular automata and are animated on the fly. The game can smoothly hold over 10000 fish sprites while only using 2 gigabytes of ram and 2 small images on the drive. The game is a progressions game where you earn money by selling fish you caught in order to buy upgrades that can catch better fish. The world also increases as you upgrade your boat. When the boat is upgraded, you have the oppertunity to move left or right further. Further from the starting point are different oceans that hold unique fish. The world wraps according to what upgrade of boat you have.

• Wrote 4500 lines of C++ code using the LARC game engine to program a responsive and in-depth video game

• Constructed upwards of 10000 individual and unique, animated fish sprites by cellular automata using procedural generation and loading

• Created terrain using procedural generation techniques by use of standard deviation

• Managed a team and tracked project version using TortoiseSVN, a subversion repository

Girl Scout Badge Tracking App

The girl scout badge tracking application is a cross platform application that allows scout leaders to easily track and manage badges or patches that scouts earn individually or as a team. Badge data is stored locally on the users device using a local database called hive. Information is persistant and holds items such as badge images, scout images, and other information.

• Wrote 3000 lines of Dart code using the Flutter framework to create a multiplatform mobile application on IOS and android devices

• Created a scalable database with 3 tables that are inner joined when presenting information
